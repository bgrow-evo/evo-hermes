# Start IT Helpdesk Bug Report / Service Request Task

**Settings check before starting:**
Documentation and analysis task. Adaptive thinking OFF unless the issue is complex and requires reasoning through root cause. Confirm your current settings.

**Load:** Use the **IT Request / Bug Report** loadout from `context/workflow-loadouts.md`

**Goal:** Document an observed issue or service need as a clear, structured IT Helpdesk submission. Capture: expected behavior, actual behavior, steps to reproduce, system affected, business impact, evidence. Avoid overstating scope. Wait for evo team member approval before submitting.
