# PIM Category Image Ordering

Load when ordering images in PIM or auditing category-specific image order, or when assigning numeric prefixes to output files during image editing.

**Always check this file before asking the evo team member about image order.** If the product category is not listed or the product type is ambiguous, flag it and ask for clarification.

*Last updated: 2026-04-28. Source: PIM Photo Guide (Google Slides, 142 slides). All ordering rules sourced from guide unless noted.*

---

## Universal rules

- Most interesting colorway to top of image matrix; black/gray/white to bottom
- Main image: white background, no lifestyle, no props
- Back shot is never 2nd — most common ordering error
- Detail images: clone them (never Shared Alternate Image)
- Product row must be at the bottom when it contains detail shots
- All items must be featured in the main image for any sets, kits, or packages
- Lifestyle images do not need to be cropped; product should be the focus
- If pulling multiple colorways for the same parent SKU and images vary, choose mains with the same angle and orientation across colorways for consistency
- No 3D rendered, vector illustration, or tech pack imagery of any kind. Exception requires manager approval.
- Templates must be used for: skis, snowboards, ski poles, skateboards/longboard completes, surfboards, wakeboards, wakesurf boards, SUP boards. Template must be hidden in the final image.

---

## Additional ordering rules

- Alt colors go BEFORE details
- Swatch details go AFTER full product flat
- Stuff sack goes LAST
- Wheelsets: both wheels in main
- Skate decks: base/graphic is the main image (NOT front facing — exception to front-facing rule)

---

## Hardgoods — image matrix set to Size

When matrix is Size (e.g. skis):
1. Drop parent EB-SKU folder → images land in Product field
2. Duplicate main image for each size → drag to each size field
3. Move Product bar to the very bottom
4. Verify detail image order in Product field
5. Save

## Bags

**Bag-specific on-model sequencing exception:** For bag products, the on-model shot where the bag is most visible takes priority. Since bags are typically worn on the back or side, the first on-model image is the back-of-model shot (bag fully visible). Apply a logical rotation as if the model is turning in place: **back → side(s) → front → full-body**. This is the reverse of the standard apparel sequence (which is front → side → back → full-body). Subjective judgment applies — the goal is to keep the bag prominent throughout.

**The most common error is placing Side after Front — Side must always come between Back and Front.**

**Detail/close-crop shots always go last** within any on-model sequence. A waist-up or cropped on-model shot must never precede a full-body on-model shot.

**Note:** A dedicated product photo taxonomy and sequencing standards file (`context/standards/product-photo-taxonomy.md`) is planned — see backlog. Until it exists, apply the rules above with human confirmation.
